# o1 / o1-mini — Official Prompt Guidance

| Field | Value |
|-------|-------|
| **Model** | `o1`, `o1-mini`, `o1-preview` |
| **Provider** | OpenAI |
| **Source** | [OpenAI Reasoning Models Guide](https://platform.openai.com/docs/guides/reasoning) |
| **Last reviewed** | 2025-04 |

---

## Key Characteristics

- **Reasoning model**: o1 performs internal chain-of-thought ("thinking") before responding. You do not see this thinking.
- **Optimised for**: complex reasoning — math, coding, scientific analysis, multi-step logic.
- **Context window**: 200 000 tokens (o1), 128 000 tokens (o1-mini).
- **No streaming** of intermediate reasoning tokens (only the final answer is returned).
- **System message support**: available in the API; use it for high-level goals and constraints.

---

## Official Guidance

### 1. Keep prompts simple and direct

Unlike GPT-4, o1 does its own internal reasoning. **Do not** pepper the prompt with chain-of-thought instructions like "think step by step" — the model already does this internally. Verbose meta-instructions can confuse or degrade output.

> ✅ "Solve this integral and return the result."
> ❌ "Think carefully step by step, list all your workings, then solve this integral."

### 2. Avoid explicit chain-of-thought prompting

- Do **not** ask the model to "think aloud", "show your work", or "reason step by step" in the user prompt.
- The model's internal reasoning is hidden; requesting it in the response wastes output tokens and can degrade accuracy.
- Exception: if you *want* a structured explanation as part of the output, ask for the final explanation format explicitly (e.g. "Provide your answer followed by a concise explanation.").

### 3. Use delimiters for structure

- Use Markdown headers, numbered lists, or XML tags to separate distinct sections (problem statement, constraints, desired output format).
- Clear structure helps the model focus its reasoning on the right sub-problem.

### 4. Specify output format precisely

- State the expected format up front: JSON schema, table, equation, plain prose.
- o1 follows format instructions reliably.

### 5. Reserve o1 for complex tasks

- For simple Q&A or text generation, GPT-4o is faster and cheaper.
- Use o1 when the task requires multi-step logical deduction, mathematical proof, competitive programming, or nuanced analysis.

### 6. Set the `reasoning_effort` parameter (API)

- `reasoning_effort: "low"` — faster, fewer thinking tokens, lower cost (suitable for moderately complex tasks).
- `reasoning_effort: "medium"` — balanced (default).
- `reasoning_effort: "high"` — maximum thinking tokens (best for hardest problems).

## Prompt Template

```
[SYSTEM]
You are <persona>. <High-level goal>. <Hard constraints>.

[USER]
Problem:
<Clear, precise description of the task>

Constraints:
- <Constraint 1>
- <Constraint 2>

Output format:
<Exact format required>
```

---

## What NOT to Do

| Anti-pattern | Why |
|-------------|-----|
| "Think step by step" | Model already reasons internally; this wastes tokens |
| "Show all your workings" | Internal reasoning is not exposed; you'll get a fabricated trace |
| Long meta-instructions about reasoning style | Degrades o1's internal search; keep prompts focused |
| Using o1 for simple tasks | Over-engineered and costly; use GPT-4o instead |
